# 🖥️ 로컬 개발 환경 설정 가이드

로컬에서 스프링 부트 서버를 실행할 때 `ENCRYPTION_KEY`와 `GEMINI_API_KEY` 환경 변수를 설정하는 방법입니다.

---

## ✅ 방법 1: IDE에서 환경 변수 설정 (권장)

### IntelliJ IDEA

1. **Run/Debug Configurations 열기**
   - 상단 메뉴: `Run` > `Edit Configurations...`
   - 또는 툴바의 실행 설정 드롭다운 > `Edit Configurations...`

2. **환경 변수 추가**
   - 왼쪽에서 실행할 설정 선택 (예: `YouthApplication`)
   - `Environment variables` 필드 찾기
   - 또는 `Modify options` > `Environment variables` 체크

3. **환경 변수 값 입력**
   ```
   ENCRYPTION_KEY=qm9MfSTvjn8IvepsCn0cAqKXLxm+Nf6da40n0kMN2e4=;GEMINI_API_KEY=iFfWIWG6SuICN30uyNLwZsh26/+DD9k7/tbBtCNSDycIve0IyNB2DrXSZTLWCC4B
   ```

   또는 각각 따로 추가:
   ```
   ENCRYPTION_KEY=qm9MfSTvjn8IvepsCn0cAqKXLxm+Nf6da40n0kMN2e4=
   GEMINI_API_KEY=iFfWIWG6SuICN30uyNLwZsh26/+DD9k7/tbBtCNSDycIve0IyNB2DrXSZTLWCC4B
   ```

4. **Apply** > **OK** 클릭

5. **애플리케이션 실행**

## ✅ 방법 2: PowerShell에서 환경 변수 설정 후 실행

### 현재 세션에서만 적용

```powershell
# 환경 변수 설정
$env:ENCRYPTION_KEY = "qm9MfSTvjn8IvepsCn0cAqKXLxm+Nf6da40n0kMN2e4="
$env:GEMINI_API_KEY = "iFfWIWG6SuICN30uyNLwZsh26/+DD9k7/tbBtCNSDycIve0IyNB2DrXSZTLWCC4B"

# 스프링 부트 실행
.\gradlew bootRun
```

또는 한 줄로:

```powershell
$env:ENCRYPTION_KEY="qm9MfSTvjn8IvepsCn0cAqKXLxm+Nf6da40n0kMN2e4="; $env:GEMINI_API_KEY="iFfWIWG6SuICN30uyNLwZsh26/+DD9k7/tbBtCNSDycIve0IyNB2DrXSZTLWCC4B"; .\gradlew bootRun
```

## ✅ 방법 3: 실행 스크립트 생성

### run-local.ps1 파일 생성

프로젝트 루트에 `run-local.ps1` 파일 생성:

```powershell
# 로컬 개발 환경 변수 설정 스크립트

Write-Host "로컬 개발 서버 시작..." -ForegroundColor Cyan

# 환경 변수 설정
$env:ENCRYPTION_KEY = "qm9MfSTvjn8IvepsCn0cAqKXLxm+Nf6da40n0kMN2e4="
$env:GEMINI_API_KEY = "iFfWIWG6SuICN30uyNLwZsh26/+DD9k7/tbBtCNSDycIve0IyNB2DrXSZTLWCC4B"

Write-Host "환경 변수 설정 완료" -ForegroundColor Green
Write-Host "ENCRYPTION_KEY: 설정됨" -ForegroundColor Gray
Write-Host "GEMINI_API_KEY: 설정됨" -ForegroundColor Gray
Write-Host ""

# 스프링 부트 실행
.\gradlew bootRun
```

실행:

```powershell
.\run-local.ps1
```

## ✅ 방법 4: application-local.yml에 직접 설정 (비권장)

**⚠️ 보안 주의**: 이 방법은 `.gitignore`에 포함되어 있지만, 가능하면 환경 변수 사용을 권장합니다.

`src/main/resources/application-local.yml` 파일 수정:

```yaml
gemini:
  api:
    key: ${GEMINI_API_KEY:암호화된_키_여기에}

# 또는 환경 변수로 직접 참조
```

그리고 시스템 속성으로 설정:

```yaml
# application-local.yml
encryption:
  key: ${ENCRYPTION_KEY:암호화_키_여기에}
```

하지만 `GeminiService`에서 `@Value("${ENCRYPTION_KEY:}")`로 직접 환경 변수를 읽고 있으므로, application.yml에 추가 설정이 필요할 수 있습니다.

## ✅ 방법 5: 시스템 환경 변수로 설정 (영구)

### Windows 시스템 환경 변수 설정

1. **시스템 속성 열기**
   - `Win + Pause` 또는 `제어판` > `시스템` > `고급 시스템 설정`

2. **환경 변수 버튼 클릭**

3. **시스템 변수에 추가**
   - `새로 만들기` 클릭
   - 변수 이름: `ENCRYPTION_KEY`
   - 변수 값: `qm9MfSTvjn8IvepsCn0cAqKXLxm+Nf6da40n0kMN2e4=`
   - `확인` 클릭

   - 다시 `새로 만들기` 클릭
   - 변수 이름: `GEMINI_API_KEY`
   - 변수 값: `iFfWIWG6SuICN30uyNLwZsh26/+DD9k7/tbBtCNSDycIve0IyNB2DrXSZTLWCC4B`
   - `확인` 클릭

4. **모든 창 닫기**

5. **IDE 재시작** (환경 변수 적용을 위해)

  - Model: gemini-2.5-flash
  - API Key: AIzaSyC2Sz... (복호화 완료)

## 🆘 문제 해결

### 환경 변수가 적용되지 않는 경우

1. **IDE 재시작** (방법 1 사용 시)
2. **PowerShell 세션 재시작** (방법 2 사용 시)
3. **환경 변수 확인**:
   ```powershell
   echo $env:ENCRYPTION_KEY
   echo $env:GEMINI_API_KEY
   ```

### 값에 특수문자가 있는 경우

특수문자(`+`, `/`, `=`)는 URL 인코딩하지 말고 그대로 입력하세요.
