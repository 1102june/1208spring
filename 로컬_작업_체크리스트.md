# 📋 로컬 작업 체크리스트

서버 배포 전 로컬에서 완료해야 할 모든 작업을 정리했습니다.

---

## ✅ 작업 순서

### 1️⃣ 프로젝트 빌드 및 컴파일 확인

```powershell
cd C:\WiseYoung_backend

# 프로젝트 빌드 (테스트 제외)
.\gradlew clean build -x test
```

**확인 사항:**
- ✅ 빌드 성공 여부
- ✅ 컴파일 오류 없음
- ✅ JAR 파일 생성: `build\libs\youth-0.0.1-SNAPSHOT.jar`

---

### 2️⃣ 새 API 키 암호화

#### 2-1. 암호화 키 생성

```powershell
# 빌드가 완료된 후
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool generate
```

**출력값을 복사하세요** → 이것이 `ENCRYPTION_KEY`입니다.

예시 출력:
```
abcd1234efgh5678ijkl9012mnop3456qrst7890uvwx...
```

#### 2-2. API 키 암호화

```powershell
# 위에서 생성한 암호화 키 사용
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool encrypt "AIzaSyC2SzM_GRKI0pr1AT4PkT7ID29hIM3hFhw" "생성된_암호화_키"
```

**출력값을 복사하세요** → 이것이 암호화된 `GEMINI_API_KEY`입니다.

예시 출력:
```
xyz7890abc1234def5678ghi9012jkl3456mno7890pqr...
```

---

### 3️⃣ 암호화 결과 저장

생성된 두 값을 안전하게 저장하세요:

```
ENCRYPTION_KEY=생성된_암호화_키
GEMINI_API_KEY=암호화된_API_키
```

**저장 방법 (선택):**
- 텍스트 파일로 저장 (Git에 커밋하지 말 것!)
- 비밀번호 관리자에 저장
- 메모장에 임시 저장 (서버 설정 후 삭제)

---

### 4️⃣ 복호화 테스트 (선택사항)

암호화가 제대로 되었는지 확인:

```powershell
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool decrypt "암호화된_API_키" "암호화_키"
```

**예상 결과:** 원본 API 키 `AIzaSyC2SzM_GRKI0pr1AT4PkT7ID29hIM3hFhw`가 출력되어야 합니다.

---

### 5️⃣ 로컬 개발 환경 설정 (선택사항)

로컬에서 개발/테스트할 계획이 있다면:

로컬 `.env` 파일 생성 (프로젝트 루트 또는 `src/main/resources/`):

```bash
# .env 파일 (로컬 개발용)
ENCRYPTION_KEY=로컬에서_사용할_암호화_키
GEMINI_API_KEY=로컬에서_사용할_암호화된_API_키
```

**중요:** `.env` 파일은 `.gitignore`에 포함되어 있어 Git에 커밋되지 않습니다.

---

### 6️⃣ 최종 빌드 및 확인

서버 배포용 최종 빌드:


- ✅ JAR 파일 생성 시간이 최근임

---

### 7️⃣ 서버 배포 준비

#### 7-1. 서버 .env 파일 내용 준비

서버 `.env` 파일에 추가할 내용:

```bash
# 기존 내용 유지하고 아래 두 줄 추가/수정

# 🔐 암호화 키 (복호화에 사용)
ENCRYPTION_KEY=2단계에서_생성한_암호화_키

# 🔐 암호화된 Gemini API 키
GEMINI_API_KEY=2단계에서_생성한_암호화된_API_키
```

#### 7-2. 배포 스크립트 확인

```powershell

# 또는
cat .\서버_접속_스크립트.ps1
```

## 🚀 다음 단계: 서버 배포

로컬 작업이 모두 완료되면:

1. **JAR 파일 업로드**
   ```powershell
   scp build\libs\youth-0.0.1-SNAPSHOT.jar root@210.104.76.139:/home/root/wiseyoung-backend/app.jar
   ```

2. **서버 .env 파일 업데이트**
   - 생성한 `ENCRYPTION_KEY`와 `GEMINI_API_KEY` 추가

3. **서비스 재시작**
   ```bash
   sudo systemctl restart wiseyoung-backend
   ```

자세한 내용은 `실제_서버_배포_가이드.md` 참조

## 🆘 문제 해결

```powershell

.\gradlew clean
.\gradlew build -x test --refresh-dependencies
```

```powershell

java -version
# Java 17 이상 필요

# 클래스패스 확인
ls build\classes\java\main\com\example\youth\util\
# ApiKeyCryptoTool.class 파일이 있어야 함
```
