# 로컬 개발 빠른 시작 가이드

## ✅ 지금 바로 로컬 개발 시작하기

### 1단계: 백엔드 서버 실행 (터미널 1)

```powershell
cd C:\Users\subpa\IdeaProjects\WiseYoung_backend
.\gradlew bootRun
```

서버가 `http://127.0.0.1:8080`에서 실행됩니다.

### 2단계: 안드로이드 앱 실행

Android Studio에서:
1. 앱 실행 버튼 (▶) 클릭
2. **자동으로 로컬 서버 (`http://127.0.0.1:8080`)에 연결됩니다**

---

## 🔍 현재 설정 확인

### 안드로이드 앱 Base URL
**현재**: `http://127.0.0.1:8080/` (로컬 서버)
- `NetworkModule.kt` 파일에서 확인 가능
- 로컬 개발 중이므로 로컬 서버 사용

### 모드 확인 방법

Android Studio에서:
1. `Build` 메뉴 → `Select Build Variant`
2. `app` 모듈의 Variant 확인
   - 보통 **`debug`**로 설정되어 있음
   - 이게 정상입니다 (로컬 개발)

## 💡 핵심 정리

### 현재 상태
- ✅ **로컬 개발 모드**로 설정되어 있습니다
- ✅ Android Studio에서 실행하면 **자동으로 로컬 서버 사용**
- ✅ 별도 설정 변경 없이 바로 개발 가능

### 개발 흐름
```
1. 로컬 백엔드 서버 실행 (http://127.0.0.1:8080)
   ↓
2. Android Studio에서 앱 실행
   ↓
3. 자동으로 로컬 서버에 연결
   ↓
4. 코드 수정하고 테스트 반복
```

---

## 🚀 개발 중 작업

### 코드 수정 후
1. 백엔드 코드 수정 → 서버 재시작 (Ctrl+C 후 다시 `.\gradlew bootRun`)
2. 안드로이드 코드 수정 → 앱 재실행 (Android Studio에서 자동)

### 서버 배포가 필요할 때
1. `NetworkModule.kt`에서 Base URL을 서버 IP로 변경
2. 서버에 배포
3. 테스트 후 다시 로컬로 변경

---

## ⚙️ 현재 설정

### NetworkModule.kt
```kotlin
private const val BASE_URL = "http://127.0.0.1:8080/" // 로컬 개발용
```

**이 설정으로 계속 개발하면 됩니다!**

배포할 때만 아래로 변경:
```kotlin
private const val BASE_URL = "http://210.104.76.139:8080/" // 서버 배포용
```

## 🆘 문제 발생 시

### 앱이 서버에 연결 안 됨
1. 백엔드 서버가 실행 중인지 확인
2. 브라우저에서 `http://127.0.0.1:8080/api/health` 접속 테스트
3. Android Studio에서 프로젝트 Clean & Rebuild

### Base URL 확인
- `app/src/main/java/com/example/app/network/NetworkModule.kt` 파일 확인
- 현재 `http://127.0.0.1:8080/`로 설정되어 있으면 정상
