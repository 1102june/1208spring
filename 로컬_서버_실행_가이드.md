# 로컬 서버 실행 가이드

## 🚀 빠른 시작 (권장)

가장 간단한 방법은 `run-local.ps1` 스크립트를 사용하는 것입니다:

```powershell
.\run-local.ps1
```

이 스크립트는 자동으로 필요한 환경 변수를 설정하고 서버를 실행합니다.

---

## 📝 수동으로 환경 변수 설정하기

`run-local.ps1` 스크립트를 사용하지 않고 직접 서버를 실행하려면, 먼저 환경 변수를 설정해야 합니다.

### Windows PowerShell

```powershell
# 현재 세션에만 적용 (PowerShell 창을 닫으면 사라짐)
# ⚠️ 보안 주의: 실제 암호화 키는 관리자에게 문의하세요.
$env:ENCRYPTION_KEY = '암호화_키_여기에_입력'
$env:GEMINI_API_KEY = '암호화된_API_키_여기에_입력'

# 서버 실행
.\gradlew bootRun
```

### Windows CMD

```cmd
# ⚠️ 보안 주의: 실제 암호화 키는 관리자에게 문의하세요.
set ENCRYPTION_KEY=암호화_키_여기에_입력
set GEMINI_API_KEY=암호화된_API_키_여기에_입력

gradlew bootRun
```

### Linux/Mac

```bash
# ⚠️ 보안 주의: 실제 암호화 키는 관리자에게 문의하세요.
export ENCRYPTION_KEY='암호화_키_여기에_입력'
export GEMINI_API_KEY='암호화된_API_키_여기에_입력'

./gradlew bootRun
```

---

## ⚠️ 오류 해결

### "ENCRYPTION_KEY 환경 변수가 설정되지 않았습니다" 오류

이 오류가 발생하면:

1. **권장 방법**: `run-local.ps1` 스크립트 사용
   ```powershell
   .\run-local.ps1
   ```

2. **수동 설정**: 위의 "수동으로 환경 변수 설정하기" 섹션 참조

3. **환경 변수 확인**:
   ```powershell
   # PowerShell에서 확인
   echo $env:ENCRYPTION_KEY
   echo $env:GEMINI_API_KEY
   ```

---

## 📌 참고사항

- 환경 변수는 **현재 터미널 세션에만** 적용됩니다.
- 새로운 터미널 창을 열면 다시 설정해야 합니다.
- 영구적으로 설정하려면 시스템 환경 변수에 추가하거나, `run-local.ps1` 스크립트를 사용하세요.
