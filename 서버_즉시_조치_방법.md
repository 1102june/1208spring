# 🚨 서버 ENCRYPTION_KEY 오류 즉시 해결 방법

## 현재 상황

서버에서 다음 오류가 발생:
```
⚠️ ERROR: API 키 복호화 실패!
오류 메시지: ENCRYPTION_KEY 환경 변수가 설정되지 않았습니다.
```

---

## ⚡ 빠른 해결 방법

### 방법 1: 서버 .env 파일에 ENCRYPTION_KEY 추가

서버에 SSH 접속 후:

```bash
# 1. 서버 접속
ssh root@210.104.76.139

# 2. .env 파일 편집
nano /home/root/wiseyoung-backend/.env
```

`.env` 파일에 다음 줄을 추가:

```bash
ENCRYPTION_KEY=여기에_암호화_키_붙여넣기
```

**중요**: 
- `ENCRYPTION_KEY`는 로컬에서 API 키를 암호화할 때 생성된 키입니다
- 아직 암호화하지 않았다면 아래 "먼저 해야 할 작업"을 참조하세요

### 방법 2: 임시 해결 (암호화 없이 사용)

**⚠️ 보안상 권장하지 않지만, 테스트용으로만 사용:**

만약 아직 API 키를 암호화하지 않았다면, 일단 평문 API 키를 사용할 수 있습니다:

```bash
# .env 파일에 추가
ENCRYPTION_KEY=임시키_사용하지_않음
GEMINI_API_KEY=AIzaSyC2SzM_GRKI0pr1AT4PkT7ID29hIM3hFhw
```

이 경우 암호화되지 않은 키가 그대로 사용됩니다 (하위 호환성).

---

## 📋 올바른 해결 순서

### 1단계: 로컬에서 API 키 암호화 (아직 안 했다면)

```powershell
# 로컬에서 실행
cd C:\WiseYoung_backend

# 빌드
.\gradlew build -x test

# 암호화 키 생성 (출력값 복사!)
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool generate

# API 키 암호화 (위에서 생성한 키 사용)
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool encrypt "AIzaSyC2SzM_GRKI0pr1AT4PkT7ID29hIM3hFhw" "생성된_암호화_키"
```

### 2단계: 서버 .env 파일 업데이트

생성된 두 값을 서버 `.env` 파일에 추가:

```bash
# 서버에서
nano /home/root/wiseyoung-backend/.env
```

다음 두 줄 추가/수정:

```bash
ENCRYPTION_KEY=1단계에서_생성한_암호화_키
GEMINI_API_KEY=1단계에서_생성한_암호화된_API_키
```

### 3단계: 서비스 재시작

```bash
# 서비스 재시작
sudo systemctl daemon-reload
sudo systemctl restart wiseyoung-backend

# 상태 확인
sudo systemctl status wiseyoung-backend

# 로그 확인 (복호화 성공 여부)
sudo journalctl -u wiseyoung-backend -n 50 --no-pager | grep -i "gemini\|복호화"
```

성공 메시지:
```
Gemini API 초기화:
  - Base URL: https://generativelanguage.googleapis.com/v1
  - Model: gemini-2.5-flash
  - API Key: AIzaSyC2Sz... (복호화 완료)
```

## 🔍 현재 서버 상태 확인

서버에서 다음 명령어로 현재 상태 확인:

```bash
# .env 파일 확인
cat /home/root/wiseyoung-backend/.env | grep -E "ENCRYPTION_KEY|GEMINI_API_KEY"

# 환경 변수 로드 확인
sudo systemctl show wiseyoung-backend --property=Environment | grep ENCRYPTION_KEY
```

---

## ⚠️ 중요 사항

1. **ENCRYPTION_KEY와 GEMINI_API_KEY 둘 다 필요**
   - 하나라도 없으면 오류 발생

2. **암호화 키는 안전하게 보관**
   - 서버 `.env` 파일에만 저장
   - Git에 커밋하지 않기

3. **서비스 재시작 필수**
   - `.env` 파일 수정 후 반드시:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl restart wiseyoung-backend
   ```

## 🆘 여전히 문제가 있는 경우

