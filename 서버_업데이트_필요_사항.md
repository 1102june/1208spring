로컬에서 빌드된 최신 JAR 파일 (암호화 기능 포함)을 서버에 업로드:

```powershell
# 로컬에서 실행
cd C:\WiseYoung_backend
scp build\libs\youth-0.0.1-SNAPSHOT.jar root@210.104.76.139:/home/root/wiseyoung-backend/app.jar
```

### 2단계: 서버 .env 파일 업데이트

서버에 SSH 접속:

```bash
ssh root@210.104.76.139
```

`.env` 파일 편집:

```bash
nano /home/root/wiseyoung-backend/.env
```

다음 두 줄을 추가/수정:

```bash
# 🔐 암호화 키 (복호화에 사용)
ENCRYPTION_KEY=TksJPfalBU2+pIU/A87y1nlswEHECggq5hS6JOOGZ2M=

# 🔐 암호화된 Gemini API 키
GEMINI_API_KEY=onZNUTFo6WhS02toy3wryzG2L8XVJiz+p1OJxbSmyeoNJv6ljk2hHRR4gfOr9RSX
```

**중요**: 
- 기존 `.env` 파일의 다른 값들은 유지
- 위 두 줄만 추가/수정

저장: `Ctrl + X`, `Y`, `Enter`

### 3단계: 서비스 재시작

```bash
# 서비스 재시작
sudo systemctl daemon-reload
sudo systemctl restart wiseyoung-backend

# 상태 확인
sudo systemctl status wiseyoung-backend

# 로그 확인 (복호화 성공 여부)
sudo journalctl -u wiseyoung-backend -n 50 --no-pager | grep -i "gemini\|복호화"
```

**성공 메시지 예시:**
```

  - Model: gemini-2.5-flash
```
