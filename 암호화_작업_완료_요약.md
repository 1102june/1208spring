# 🔐 Gemini API 키 암호화 구현 완료

## ✅ 완료된 작업

### 1. 암호화 유틸리티 클래스
- **파일**: `src/main/java/com/example/youth/util/CryptoUtil.java`
- **기능**: AES-256 암호화/복호화
- **사용법**: `CryptoUtil.encrypt(plainText, key)`, `CryptoUtil.decrypt(encryptedText, key)`

### 2. GeminiService 수정
- **파일**: `src/main/java/com/example/youth/service/GeminiService.java`
- **변경사항**:
  - 암호화된 API 키를 환경 변수에서 읽기
  - `ENCRYPTION_KEY`로 복호화하여 사용
  - 초기화 시 복호화 상태 로그 출력

### 3. 암호화 도구
- **파일**: `src/main/java/com/example/youth/util/ApiKeyCryptoTool.java`
- **기능**: 명령줄 암호화 도구
  - 키 생성: `generate`
  - 암호화: `encrypt <평문> <키>`
  - 복호화: `decrypt <암호화> <키>`

### 4. PowerShell 스크립트
- **파일**: `API_키_암호화_스크립트.ps1`
- **기능**: 자동화된 암호화 프로세스

### 5. 문서
- `API_키_보안_가이드.md` - 보안 모범 사례
- `API_키_암호화_가이드.md` - 상세 사용 가이드
- `API_키_암호화_간단_방법.md` - 빠른 시작

## 🚀 다음 단계: 로컬에서 작업

### 1. 새 API 키 암호화

```powershell
# 프로젝트 디렉토리로 이동
cd C:\WiseYoung_backend

# 빌드
.\gradlew build -x test

# 암호화 키 생성 (출력값 복사!)
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool generate

# API 키 암호화 (위에서 생성한 키 사용)
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool encrypt "AIzaSyC2SzM_GRKI0pr1AT4PkT7ID29hIM3hFhw" "생성된_암호화_키"
```

### 2. 결과 확인

두 개의 값이 생성됩니다:
- **ENCRYPTION_KEY**: 복호화에 사용할 키
- **GEMINI_API_KEY**: 암호화된 API 키

### 3. 테스트 (선택사항)

복호화가 잘 되는지 확인:

```powershell
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool decrypt "암호화된_키" "암호화_키"
```

원본 API 키가 출력되면 성공!

## ⚠️ 중요 사항

1. **암호화 키는 절대 노출 금지**
   - 서버 `.env` 파일에만 저장
   - Git에 커밋하지 않기
   - 문서에 실제 키 포함하지 않기

2. **암호화 키 백업 필수**
   - 키를 잃어버리면 복호화 불가

3. **서버 설정 필수**
   - `ENCRYPTION_KEY`와 `GEMINI_API_KEY` 둘 다 설정 필요
   - 하나라도 없으면 애플리케이션 시작 실패
