# 🔌 로컬 서버 연결 가이드

## 현재 상황 분석

로그를 보면:
- ✅ Google 로그인 성공
- ✅ Firebase 인증 성공  
- ✅ 서버에 요청 전송 및 응답 수신 (404)
- ✅ 프로필이 없어서 ProfileSetupActivity로 이동

**이것은 정상적인 동작입니다!** 404는 "사용자를 찾을 수 없습니다"라는 의미로, 처음 로그인하는 사용자에게는 정상입니다.

---

## ⚠️ 중요: BASE_URL 설정

### 실제 디바이스나 에뮬레이터 사용 시

`127.0.0.1`은 **PC 자체**를 가리키므로, 안드로이드 디바이스/에뮬레이터에서는 접근할 수 없습니다!

#### 안드로이드 에뮬레이터 사용 시

`NetworkModule.kt`에서:

```kotlin
private const val BASE_URL = "http://10.0.2.2:8080/" // 안드로이드 에뮬레이터용
```

`10.0.2.2`는 에뮬레이터에서 호스트 PC를 가리키는 특수 주소입니다.

#### 실제 디바이스 사용 시

PC의 로컬 IP 주소를 사용해야 합니다:

1. **PC의 IP 주소 확인**:
   ```powershell
   ipconfig
   ```
   `IPv4 주소`를 찾으세요 (예: `192.168.0.100`)

2. **NetworkModule.kt 수정**:
   ```kotlin
   private const val BASE_URL = "http://192.168.0.100:8080/" // 실제 디바이스용 (PC의 IP)
   ```

3. **PC와 디바이스가 같은 Wi-Fi 네트워크에 연결되어 있어야 합니다**

## 🔍 로컬 서버 연결 확인

### 1. 로컬 서버가 실행 중인지 확인

PowerShell에서:

```powershell
# 서버가 실행 중인지 확인
netstat -an | findstr "8080"
```

또는 브라우저에서:
```
http://localhost:8080/api/health
```

### 2. 서버 로그 확인

로컬 서버의 콘솔에서 요청이 들어오는지 확인하세요.

## 📱 테스트 환경별 설정

### 시나리오 1: 안드로이드 에뮬레이터

```kotlin
private const val BASE_URL = "http://10.0.2.2:8080/"
```

### 시나리오 2: 실제 디바이스 (같은 Wi-Fi)

```kotlin
private const val BASE_URL = "http://192.168.x.x:8080/" // PC의 IP
```

### 시나리오 3: USB 디버깅 (ADB 포트 포워딩)

```kotlin
private const val BASE_URL = "http://127.0.0.1:8080/"
```

그리고 ADB 포트 포워딩:
```bash
adb reverse tcp:8080 tcp:8080
```

## ✅ 현재 로그 분석

로그에서 확인된 내용:
- `프로필 확인 응답 코드: 404` → 서버에 연결은 성공했지만 프로필이 없음
- `프로필 없음 플래그 설정 - ProfileSetupActivity로 이동` → 정상 플로우

**결론**: 서버 연결은 정상입니다! 404는 예상된 응답입니다.
