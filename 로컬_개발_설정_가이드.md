# 로컬 개발 환경 설정 가이드

## 🎯 현재 상태 확인

### Android Studio에서 실행할 때
- **Debug 모드**: Android Studio에서 직접 실행 (▶ 버튼) → 자동으로 Debug 빌드
- **Release 모드**: `assembleRelease`로 빌드할 때만

### 현재 설정 확인 방법

1. **안드로이드 앱 실행 중 Logcat 확인**
   ```
   NetworkModule에서 Base URL 확인
   ```

2. **간단한 확인 방법**
   - Debug 모드: 로컬 서버 (`http://127.0.0.1:8080/`) 사용
   - Release 모드: 서버 IP (`http://210.104.76.139:8080/`) 사용

```bash
cd C:\Users\subpa\IdeaProjects\WiseYoung_backend
```

또는 IntelliJ IDEA에서:
1. `YouthApplication.java` 실행
2. 서버가 `http://127.0.0.1:8080`에서 실행되는지 확인

### 3. 안드로이드 앱 실행

Android Studio에서:
1. 앱 실행 (▶ 버튼) → **자동으로 Debug 모드**
2. 로그 확인 → `http://127.0.0.1:8080/` 연결 확인

## 🔍 현재 모드 확인 방법

### 방법 1: Logcat 확인

앱 실행 후 Logcat에서:
```
NetworkModule: Base URL = http://127.0.0.1:8080/
```
→ Debug 모드 (로컬 서버)

또는
```
NetworkModule: Base URL = http://210.104.76.139:8080/
```
→ Release 모드 (서버)

### 방법 2: 빌드 Variant 확인

Android Studio에서:
1. `Build` → `Select Build Variant`
2. `app` 모듈의 Variant 확인
   - `debug` → Debug 모드 (로컬 서버)
   - `release` → Release 모드 (서버)

## 🛠️ 로컬 개발 설정 (명시적)

만약 BuildConfig가 제대로 작동하지 않는다면, 임시로 명시적으로 설정할 수 있습니다:

### 옵션 1: BuildConfig 사용 (현재 설정 - 권장)

이미 설정되어 있으므로, Android Studio에서 실행하면 자동으로 로컬 서버 사용

### 옵션 2: 임시로 명시적 설정

로컬 개발을 확실히 하려면:

```kotlin
object NetworkModule {
    // 로컬 개발용 (명시적)
    private const val BASE_URL = "http://127.0.0.1:8080/"
    
    // 또는 BuildConfig 사용 (자동)
    // private const val BASE_URL = BuildConfig.BASE_URL
}
```

## 💡 팁

### Debug 모드 확인
- Android Studio에서 직접 실행 → Debug
- `assembleDebug` 빌드 → Debug
- `assembleRelease` 빌드 → Release

### 문제 발생 시
1. Build Variant 확인: `Build` → `Select Build Variant` → `debug` 선택
2. 프로젝트 Clean & Rebuild: `Build` → `Clean Project` → `Rebuild Project`
3. 로컬 서버 확인: `http://127.0.0.1:8080/api/health` 접속 테스트

## 📞 문제 해결

### BuildConfig를 찾을 수 없다는 오류
→ 프로젝트를 Clean & Rebuild 하세요.

### 로컬 서버에 연결 안 됨
1. 백엔드 서버가 실행 중인지 확인
2. `http://127.0.0.1:8080/api/health` 브라우저에서 접속 테스트
3. 방화벽 확인
