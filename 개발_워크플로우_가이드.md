## 🔄 개발 워크플로우 전략

### 방법 1: 로컬 개발 → 주기적 서버 배포 (권장)

#### 작업 흐름
```
1. 로컬에서 개발 및 테스트
   ↓
2. 기능 완성되면 로컬에서 테스트
   ↓
3. 서버에 배포하여 실제 환경 테스트
   ↓
4. 문제 발견 시 로컬로 돌아가서 수정
   ↓
5. 다시 서버 배포
```

#### 장점
- ✅ 로컬에서 빠르게 개발 가능
- ✅ 서버는 안정적인 버전만 배포
- ✅ 서버 리소스 절약

#### 단점
- ⚠️ 서버 업데이트가 수동으로 필요
- ⚠️ 배포 시간 소요

---

### 방법 2: Git 기반 배포 (더 권장)

#### 작업 흐름
```
1. 로컬에서 개발
   ↓
2. Git에 커밋 (dev 브랜치)
   ↓
3. 기능 완성 시 main 브랜치로 머지
   ↓
4. 서버에서 Git pull
   ↓
5. 서버에서 빌드 및 재시작
```

#### 장점
- ✅ 버전 관리
- ✅ 코드 변경 이력 추적
- ✅ 롤백 가능
- ✅ 협업 용이

---

### 방법 3: 개발 서버 + 프로덕션 서버 분리

#### 작업 흐름
```
로컬 개발
   ↓
개발 서버 배포 (테스트)
   ↓
문제 해결
   ↓
프로덕션 서버 배포 (안정 버전)
```

현재는 서버가 하나이므로, 이 방법은 서버를 두 개 쓸 때 사용.

## 🚀 추천 워크플로우 (현재 상황)

### 일상적인 개발 흐름

#### 1️⃣ 로컬에서 개발
```bash
# 로컬에서 작업
- 코드 수정
- 로컬 서버 실행 (http://127.0.0.1:8080)
- 로컬에서 테스트
```

#### 2️⃣ 안드로이드 앱은 로컬 서버로 연결
**개발 중일 때**: `NetworkModule.kt`를 로컬로 변경
```kotlin
private const val BASE_URL = "http://127.0.0.1:8080/" // 개발용
```

#### 3️⃣ 기능 완성 후 서버 배포
```bash
# 1. 안드로이드 앱 Base URL 변경 (서버 IP로)
private const val BASE_URL = "http://210.104.76.139:8080/"

# 2. 로컬에서 빌드
.\빌드_스크립트.ps1

# 3. 서버에 업로드
.\서버_접속_스크립트.ps1
# 또는
scp build\libs\youth-0.0.1-SNAPSHOT.jar root@210.104.76.139:/home/root/wiseyoung-backend/app.jar

# 4. 서버에서 재시작
ssh root@210.104.76.139
sudo systemctl restart wiseyoung-backend
```

#### 4️⃣ 서버에서 테스트
- 실제 서버 환경에서 테스트
- 문제 발견 시 로컬로 돌아가서 수정

## 💡 효율적인 개발 팁

### 1. Base URL을 쉽게 전환하기

`NetworkModule.kt`를 다음과 같이 수정:

```kotlin
object NetworkModule {
    // 개발/배포 환경 자동 전환
    private const val IS_DEVELOPMENT = BuildConfig.DEBUG
    private const val DEV_BASE_URL = "http://127.0.0.1:8080/"
    private const val PROD_BASE_URL = "http://210.104.76.139:8080/"
    
    private const val BASE_URL = if (IS_DEVELOPMENT) {
        DEV_BASE_URL
    } else {
        PROD_BASE_URL
    }
    
    // ... 나머지 코드
}
```

**주의**: `build.gradle.kts`에 `buildConfig = true` 추가 필요

### 2. 자동 배포 스크립트 활용

`서버_접속_스크립트.ps1`을 사용하면:
- 빌드 + 업로드 + 재시작을 한 번에
- 자주 배포해도 편리

### 3. 서버 로그 실시간 모니터링

```bash
# 서버에서
sudo journalctl -u wiseyoung-backend -f

# 별도 터미널에서 로그 확인하면서 테스트
```

## 🔧 실전 예시

### 예시 1: 북마크 기능 수정

```
1. 로컬에서 NetworkModule.kt 확인
   → BASE_URL = "http://127.0.0.1:8080/"

2. 로컬 서버 실행
   → 백엔드 코드 수정
   → 로컬에서 테스트

3. 안드로이드 앱에서 테스트
   → 로컬 서버와 연결되어 있음
   → 문제 발견 및 수정

4. 기능 완성
   → NetworkModule.kt를 서버 IP로 변경
   → 빌드 및 배포
   → 서버에서 최종 테스트
```

### 예시 2: 서버에서만 발생하는 문제

```
1. 서버 로그 확인
   ssh root@210.104.76.139
   sudo journalctl -u wiseyoung-backend -f

2. 문제 파악 후 로컬에서 수정

3. 수정 완료 후 다시 배포
```
