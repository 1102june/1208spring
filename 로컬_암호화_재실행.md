# 🔄 로컬 암호화 재실행 가이드

복호화 실패 오류가 발생했습니다. 올바른 키로 다시 암호화하겠습니다.

---

## 문제 원인

복호화 오류: `Given final block not properly padded`

이는 암호화 키와 암호화된 키가 일치하지 않을 때 발생합니다.

---

## 해결 방법: 다시 암호화하기

### 1단계: 새 암호화 키 생성

```powershell
cd C:\WiseYoung_backend
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool generate
```

**출력된 키를 복사하세요** → 이것이 새 `ENCRYPTION_KEY`입니다.

### 2단계: API 키 암호화

```powershell
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool encrypt "AIzaSyC2SzM_GRKI0pr1AT4PkT7ID29hIM3hFhw" "1단계에서_생성한_새_암호화_키"
```

**출력된 값을 복사하세요** → 이것이 새 암호화된 `GEMINI_API_KEY`입니다.

### 3단계: 복호화 테스트

올바르게 암호화되었는지 확인:

```powershell
java -cp "build/classes/java/main" com.example.youth.util.ApiKeyCryptoTool decrypt "2단계에서_생성한_암호화된_키" "1단계에서_생성한_암호화_키"
```

**예상 결과**: `AIzaSyC2SzM_GRKI0pr1AT4PkT7ID29hIM3hFhw`가 출력되어야 합니다.

### 4단계: 환경 변수 업데이트

생성된 두 값을 사용하여 환경 변수 설정:

```powershell
$env:ENCRYPTION_KEY = '새로_생성한_암호화_키'
$env:GEMINI_API_KEY = '새로_생성한_암호화된_API_키'
```

### 5단계: 서버 재시작

```powershell
.\gradlew bootRun
```

---

## ⚠️ 중요

- 암호화 키와 암호화된 키는 **반드시 같은 세션에서 생성**해야 합니다
- 하나를 잃어버리면 복호화할 수 없습니다
- 새로 생성한 키를 안전하게 저장하세요
